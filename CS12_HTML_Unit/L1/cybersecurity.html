Cybersecurity and HTML: 5 Historical Cases

1. The Samy Worm (2005)

A MySpace user named Samy Kamkar exploited a cross-site scripting (XSS) vulnerability by injecting JavaScript into his HTML profile. Within 20 hours, over one million people had unknowingly executed his code and added him as a friend. It remains one of the fastest-spreading pieces of malicious code in history.

2. The British Airways Data Breach (2018)

Attackers injected 22 lines of malicious JavaScript into British Airways' payment page HTML, silently harvesting credit card details from roughly 500,000 customers. This attack, known as "formjacking," showed how a tiny, invisible change to an HTML page can have massive real-world consequences.

3. Tabnapping (Coined 2010)

This attack exploits user inattention to open browser tabs by using a malicious script to replace an idle tab with a fake login page for a trusted site like Gmail. It is a powerful teaching tool because the attack can work even without JavaScript, using the HTML meta refresh tag to redirect inactive tabs.

4. SQL Injection (First Documented 1998)

SQL Injection occurs when an attacker uses an HTML form input to "inject" malicious database queries into a website's backend. This allows hackers to view, change, or delete sensitive data that they aren't supposed to access, and it remains one of the most common web vulnerabilities today.
